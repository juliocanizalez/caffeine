# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |
| Older   | ❌        |

Only the latest release receives security fixes. Please upgrade before reporting.

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Report vulnerabilities privately by emailing **juliocanizalez@outlook.com** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Your suggested fix (optional)

You'll receive a response within 48 hours. If confirmed, a patch will be released as soon as possible and you'll be credited in the release notes (unless you prefer to remain anonymous).
